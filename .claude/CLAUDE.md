# Claude Code Configuration for ViFoodLabel

## Project Overview

**ViFoodLabel** is a dataset project for Vietnamese food product label Key Information Extraction (KIE), released as a **Data in Brief (DIB) data article** with the dataset deposited on Mendeley Data. This workspace contains:

- Raw images + Label Studio annotations (in `data/`)
- Data construction, validation, statistics, anonymization, and release scripts (in `scripts/`)
- Dataset specifications, annotation guidelines, and the data dictionary (in `docs/`)
- Planning/progress materials (in `.plan/`, git-ignored, local use only)

## Key Files

| File/Folder | Purpose |
|---|---|
| `docs/` | Public documentation: dataset overview, statistics, data dictionary, annotation guidelines, entity/relation schemas |
| `scripts/preprocessing/` | Image prep, OCR pre-annotation, Label Studio → token-level conversion, validation |
| `scripts/` | Task 3 KIE record + per-image metadata assembly, LLM-assisted QC linter |
| `scripts/dataset/` | Stratified splitting, statistics/figures, anonymization, EXIF stripping, packaging, checksums |
| `.plan/` | (Git-ignored) Local planning, progress tracking, and project phases — not part of public repo |
| `data/` | (Git-ignored) Images, annotations, and derived records — published separately via Mendeley Data |

## Important Notes for Assistant Work

1. **No AI Attribution**: This is an academic research repository. Remove or do not add any markers like "Co-Authored-By Claude", "generated by AI", etc. — use standard Python/Markdown conventions only.
2. **Dataset-Focused Documentation**: Public docs (in `docs/`) describe the dataset only — construction, schema, and statistics — with no baseline or benchmark experiments. Planning/progress (in `.plan/`) is strictly local and not committed.
3. **Data in Brief scope**: The repository is the data-article companion (dataset plus construction and description tooling). It has no model training, benchmarking, or serving layer.
4. **Target Venue**: Data in Brief (Elsevier) data article; dataset on Mendeley Data. Code and docs should reflect publication standards.

## File Conventions

- **Docs**: All in English except `-vi` suffix files (e.g., `annotation-vi.md` is Vietnamese).
- **Code**: Standard Python with minimal comments (only document non-obvious logic).
- **Markdown**: Plain Markdown; no HTML or custom formatting unless necessary.

## Commit Message Conventions

Follow [Conventional Commits](https://www.conventionalcommits.org/):

- **Format**: `<type>: <short description>`
- **Types**: `feat`, `fix`, `data`, `refactor`, `docs`, `test`, `chore`
- **Subject line**: Under 72 characters
- **Body** (optional, line 2+): More detailed explanation if needed

**Important**: Do NOT include data statistics (sample counts, token counts, entity distributions, etc.) in commit messages. Only describe *what* changed and *why*.

**Examples**:
- ✓ `feat: add stratified train/dev/test split script`
- ✓ `docs: document the per-image KIE record schema`
- ✗ `data: processed 416 images, 90% primary-entity coverage`

## Quickstart for Claude
